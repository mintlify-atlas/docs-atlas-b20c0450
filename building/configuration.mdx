## Configuration File

The build system uses `bootstrap.toml` for configuration. You can:

1. Create it manually
2. Generate it with `./configure`
3. Use `./x.py setup` for interactive configuration

The build system looks for `bootstrap.toml` in the current directory, or you can specify a custom file with `--config`.

<Note>
See `bootstrap.example.toml` in the repository root for a complete reference with all available options and their default values.
</Note>

## Configuration Profiles

Profiles provide pre-configured defaults optimized for different use cases:

<Tabs>
  <Tab title="library">
    Optimized for contributing to the standard library:

    ```toml
    profile = "library"
    ```

    - Downloads pre-built compiler
    - Fast iteration on library code
    - Minimal build times
  </Tab>

  <Tab title="compiler">
    Optimized for compiler development:

    ```toml
    profile = "compiler"
    ```

    - Enables debug assertions
    - Includes compiler documentation
    - Better error messages for debugging
  </Tab>

  <Tab title="codegen">
    Optimized for code generation work:

    ```toml
    profile = "codegen"
    ```

    - Focused on LLVM and codegen
    - Enables relevant debugging features
  </Tab>

  <Tab title="tools">
    Optimized for tools like Clippy and rustfmt:

    ```toml
    profile = "tools"
    ```

    - Builds extended toolset
    - Enables extra checks
  </Tab>

  <Tab title="user">
    General-purpose development:

    ```toml
    profile = "user"
    ```

    - Balanced configuration
    - Good for experimentation
  </Tab>
</Tabs>

## LLVM Configuration

LLVM is the most time-consuming part of the build. Optimize it carefully:

<AccordionGroup>
  <Accordion title="Download Pre-Built LLVM (Recommended)">
    **Fastest option** - Skip building LLVM entirely:

    ```toml
    [llvm]
    # Always download if available
    download-ci-llvm = true

    # Download only if LLVM sources haven't changed
    download-ci-llvm = "if-unchanged"
    ```

    <Tip>
    After enabling this, deinitialize the LLVM submodule to save disk space:
    ```bash
    git submodule deinit src/llvm-project
    ```
    </Tip>

    **Requirements:**
    - Only works for tier 1 and tier 2 platforms with host tools
    - Only for build triple (not cross-compilation)
  </Accordion>

  <Accordion title="Build LLVM from Source">
    If you need to build LLVM (e.g., for local LLVM changes):

    ```toml
    [llvm]
    download-ci-llvm = false

    # Use Ninja (much faster than Make)
    ninja = true

    # Optimize LLVM
    optimize = true

    # Enable assertions for better error messages
    assertions = false

    # Select targets (reduces build time)
    targets = "X86;ARM;AArch64"

    # Link LLVM dynamically (faster linking)
    link-shared = true
    ```

    <Warning>
    Building LLVM requires significant time (1-3 hours) and disk space (10+ GB).
    </Warning>
  </Accordion>

  <Accordion title="Use System LLVM">
    Use LLVM installed on your system:

    ```toml
    [target.x86_64-unknown-linux-gnu]
    llvm-config = "/usr/bin/llvm-config"
    ```

    Verify it's a Rust-patched version or set:
    ```toml
    llvm-has-rust-patches = false
    ```
  </Accordion>

  <Accordion title="Advanced LLVM Options">
    ```toml
    [llvm]
    # ThinLTO for LLVM itself (requires clang)
    thin-lto = true

    # Enable specific experimental targets
    experimental-targets = "AVR;M68k"

    # Limit parallel linker invocations (saves memory)
    link-jobs = 2

    # Include debug info in release builds
    release-debuginfo = false

    # Enable LLVM tests (for LLVM development)
    tests = false

    # Use zstd compression
    libzstd = true

    # Custom compiler flags
    cflags = "-march=native"
    cxxflags = "-march=native"
    ldflags = ""

    # Use libc++ instead of libstdc++
    use-libcxx = false

    # Specify linker for LLVM
    use-linker = "/usr/bin/lld"
    ```
  </Accordion>
</AccordionGroup>

## Build Options

Control the build process:

```toml
[build]
# Build triple (auto-detected by default)
build = "x86_64-unknown-linux-gnu"

# Host triples (platforms to build a compiler for)
host = ["x86_64-unknown-linux-gnu"]

# Target triples (platforms to build libraries for)
target = [
  "x86_64-unknown-linux-gnu",
  "aarch64-unknown-linux-gnu",
]

# Build directory
build-dir = "build"

# Parallel jobs (0 = number of CPUs)
jobs = 0

# Build extended tools
extended = true

# Specific tools to build
tools = [
  "cargo",
  "clippy",
  "rustdoc",
  "rustfmt",
  "rust-analyzer",
  "analysis",
  "src",
]

# Use vendored dependencies
vendor = false

# Locked dependencies (don't update Cargo.lock)
locked-deps = false

# Build documentation
docs = true

# Build compiler documentation
compiler-docs = false

# Manage git submodules automatically
submodules = true

# Print timing information
print-step-timings = true

# Use ccache
ccache = true
# Or specify path
# ccache = "/usr/bin/ccache"

# Low priority build (Unix: nice +10, Windows: low priority)
low-priority = false

# Enable metrics collection
metrics = false
```

<Tip>
For faster development builds, disable documentation and use ccache:

```toml
[build]
docs = false
compiler-docs = false
ccache = true
```
</Tip>

## Rust Compiler Options

Configure how the Rust compiler itself is built:

<CodeGroup>
```toml Optimization
[rust]
# Optimization level (true = 3, false = 0, 1-3, "s", "z")
optimize = true

# Code generation units (higher = faster compile, slower runtime)
codegen-units = 16

# LTO for rustc ("thin-local", "thin", "fat", "off")
lto = "thin-local"

# Strip symbols
strip = false
```

```toml Debugging
[rust]
# Debug mode (enables assertions and debug info)
debug = false

# Debug assertions
debug-assertions = true

# Debug assertions for std only
debug-assertions-std = true

# Debug logging (trace!, debug! macros)
debug-logging = false

# Debuginfo level (0-2)
debuginfolevel = 1

# Separate debuginfo levels
debuginfolevel-rustc = 1
debuginfolevel-std = 1
debuginfolevel-tools = 1
debuginfolevel-tests = 0
```

```toml Features
[rust]
# Release channel ("dev", "nightly", "beta", "stable")
channel = "dev"

# Incremental compilation
incremental = false

# Overflow checks
overflow-checks = false

# Backtrace support
backtrace = true

# Jemalloc allocator
jemalloc = false

# Build profiler runtime
profiler = false

# Build sanitizer runtimes
sanitizers = false

# Randomize type layout (for testing)
randomize-layout = false

# Additional rustflags
rustflags = ["-Cforce-frame-pointers=yes"]
```

```toml Codegen Backends
[rust]
# Codegen backends ("llvm", "cranelift", "gcc")
codegen-backends = ["llvm"]

# Build LLD
lld = false

# Use LLD for bootstrap builds
bootstrap-override-lld = false

# Include LLVM tools
llvm-tools = true
```
</CodeGroup>

## Target-Specific Configuration

Configure settings for specific target triples:

```toml
[target.x86_64-unknown-linux-gnu]
# C compiler
cc = "gcc"

# C++ compiler
cxx = "g++"

# Archiver
ar = "ar"

# Ranlib
ranlib = "ranlib"

# Linker
linker = "gcc"

# LLVM config
llvm-config = "/usr/bin/llvm-config"

# Additional rustflags for this target
rustflags = ["-Ctarget-cpu=native"]

# Static or dynamic CRT
crt-static = false

# Split debuginfo ("off", "unpacked", "packed")
split-debuginfo = "off"

# Build sanitizers for this target
sanitizers = true

# Build profiler for this target
profiler = true

# Enable rpath
rpath = true

# Test runner (for cross-compilation)
runner = "qemu-aarch64"

# Skip building std
no-std = false

# Musl root (for musl targets)
musl-root = "/usr/local/musl"

# WASI root (for WASM targets)
wasi-root = "/path/to/wasi-sdk"
```

### Cross-Compilation Example

```toml
[build]
build = "x86_64-unknown-linux-gnu"
target = ["aarch64-unknown-linux-gnu"]

[target.aarch64-unknown-linux-gnu]
cc = "aarch64-linux-gnu-gcc"
cxx = "aarch64-linux-gnu-g++"
linker = "aarch64-linux-gnu-gcc"
ar = "aarch64-linux-gnu-ar"
ranlib = "aarch64-linux-gnu-ranlib"
```

## Installation Configuration

Configure where `x.py install` places files:

```toml
[install]
# Installation prefix
prefix = "/usr/local"

# System configuration directory
sysconfdir = "/etc"

# Documentation directory
docdir = "share/doc/rust"

# Binary directory
bindir = "bin"

# Library directory
libdir = "lib"

# Man page directory
mandir = "share/man"

# Data directory
datadir = "share"
```

## Distribution Options

For building release distributions:

```toml
[dist]
# Sign artifacts
sign-folder = "/path/to/artifacts"

# Upload URL
upload-addr = "https://example.com/dist"

# Build source tarball
src-tarball = true

# Compression formats
compression-formats = ["gz", "xz"]

# Compression profile ("fast", "balanced", "best")
compression-profile = "fast"

# Include MinGW linker (Windows)
include-mingw-linker = true

# Vendor dependencies
vendor = true
```

## Using the Configure Script

The `./configure` script provides a command-line interface to generate `bootstrap.toml`:

<CodeGroup>
```bash Basic Usage
./configure --prefix=/opt/rust
```

```bash Enable Features
./configure \
  --enable-full-tools \
  --enable-profiler \
  --enable-sanitizers \
  --enable-compiler-docs
```

```bash Set Options
./configure \
  --set llvm.download-ci-llvm=true \
  --set rust.channel=nightly \
  --set build.extended=true
```

```bash Complex Example
./configure \
  --build=x86_64-unknown-linux-gnu \
  --enable-full-tools \
  --enable-profiler \
  --set target.aarch64-unknown-linux-gnu.linker=clang \
  --set llvm.link-shared=true \
  --set llvm.thin-lto=true \
  --set rust.debug-assertions=false \
  --set rust.lto=thin
```
</CodeGroup>

## Configuration Tips

<Tip>
**For library development:**
- Use `profile = "library"`
- Set `llvm.download-ci-llvm = true`
- Set `rust.download-rustc = "if-unchanged"`
</Tip>

<Tip>
**To reduce disk usage:**
- Set `llvm.download-ci-llvm = true`
- Run `git submodule deinit src/llvm-project`
- Set `build.docs = false`
</Tip>

<Tip>
**To speed up builds:**
- Use `llvm.ninja = true`
- Enable `build.ccache = true`
- Set `build.docs = false`
- Use `llvm.download-ci-llvm = true`
- Enable `rust.incremental = true`
</Tip>

<Tip>
**To reduce memory usage:**
- Set `build.jobs = 1` or a low number
- Set `llvm.link-jobs = 1`
- Use `llvm.download-ci-llvm = true`
</Tip>
